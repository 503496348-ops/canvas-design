# Presenter Mode：演讲者模式与逐字稿

用于线下分享、课程、路演、技术分享等“人要站在屏幕前讲”的场景。目标不是让幻灯片更炫，而是让演讲者**不慌、不忘、不超时**。

## 触发条件

用户出现以下任一信号，必须考虑启用 Presenter Mode：

- 演讲 / 分享 / 路演 / 线下讲 / 给团队讲
- 讲稿 / 逐字稿 / speaker notes / presenter view / 提词器
- 计时器 / 控制节奏 / 不想讲乱
- 需要当前页 + 下一页预览

## 交付物

HTML Deck 交付时额外包含：

1. 每页 `<aside class="notes">...</aside>`：150–300 字口语化逐字稿或提示稿。
2. `assets/presenter-runtime.js`：按 `S` 打开讲者窗口。
3. 讲者窗口四区：当前页、下一页、讲稿、计时器。
4. 双窗口同步：主窗口翻页时讲者窗口同步，讲者窗口翻页时主窗口同步。

## 逐字稿规则

1. **不是论文，是提示信号。** 用口语，不写“综上所述”这种书面腔。
2. **每页 150–300 字。** 少于 100 字容易断片，多于 350 字演讲者会读稿。
3. **核心词加粗或分行。** 让眼睛一扫就能抓住重点。
4. **每页要有过渡句。** 下一页为什么出现，要在最后一句铺垫。
5. **避免把逐字稿显示在观众页。** notes 只给讲者窗口读取。

## HTML 约定

```html
<section class="slide active" data-title="问题不是没内容，是没结构">
  <h1>问题不是没内容，是没结构</h1>
  <p>这一页给观众看的内容。</p>
  <aside class="notes">
    开场先别急着讲工具。这里要先把痛点讲清楚：很多团队不是缺资料，
    而是资料散在 PDF、视频、网页和聊天记录里，最后没人能把它变成一套可讲的故事。
    这一页讲完后，顺到下一页：所以我们需要的不是又一个模板，而是一条生产线。
  </aside>
</section>
<script src="assets/presenter-runtime.js"></script>
```

## 快捷键

| 键 | 行为 |
|---|---|
| ← / → | 上一页 / 下一页 |
| S | 打开或聚焦讲者窗口 |
| R | 讲者窗口重置计时器 |
| Esc | 关闭讲者窗口 |

## 质量门禁

- 主窗口和讲者窗口页码一致。
- 当前页/下一页预览正确，不闪烁。
- notes 不出现在观众页截图/PDF 中。
- 计时器可重置。
- 无 BroadcastChannel 时仍可单窗口工作，不崩。
