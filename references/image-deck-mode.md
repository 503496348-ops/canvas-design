# Image Deck Mode：图片型社交幻灯片

Image Deck 是“阅读型/传播型”幻灯片：每页是一张完整图片，适合手机阅读、社交转发、公众号配图、课程知识卡片。它不是现场演讲 PPT，也不追求 PowerPoint 内可编辑。

## 适用场景

- 小红书图文 / 朋友圈组图 / 公众号长图卡片
- 知识卡片、读书笔记、课程复习卡
- 产品卖点组图、活动宣发图
- 用户明确说“一页一张图”“发图文”“做成卡片”

## 不适用场景

- 客户需要在 PPT 里继续编辑文字 → 用 Editable PPTX Mode
- 线下分享需要讲者备注/计时器 → 用 HTML Deck + Presenter Mode
- 强交互演示 → 用 HTML Deck / Prototype

## 工作流

1. **内容切片**：每页只承载一个核心判断，标题必须是断言句。
2. **风格锁定**：先选一套视觉方向，所有页保持统一色彩/字体/图形语言。
3. **Prompt 留痕**：每页图片生成前写 `prompts/NN-slide-name.md`，便于复现和换后端。
4. **批量生成**：有图像后端时按 4 张一批；无后端时降级为 HTML/CSS 渲染图片，但必须说明不是 AI raster。
5. **合并交付**：输出 PNG/JPG 文件夹，可选合并 PDF/PPTX（每页一张图片）。

## 页面规格

| 渠道 | 推荐尺寸 | 说明 |
|---|---|---|
| 小红书图文 | 1242×1660 / 1080×1440 | 4:3 竖图，标题大，正文少 |
| 朋友圈组图 | 1080×1080 | 方图，信息密度低 |
| 公众号配图 | 1200×675 / 900×1200 | 横封面或竖卡片 |
| 通用知识卡 | 1440×1080 | 4:3，兼顾投屏和手机 |

## 文字密度

- 标题：不超过 22 个中文字符。
- 正文：每页不超过 80–120 字。
- 列表：不超过 4 条，每条不超过 18 字。
- 引用/金句页可以只有一句话，但必须有视觉张力。

## 质量门禁

- 手机宽度缩略图下仍能读标题。
- 同组图片统一边距、字号、页码和品牌角标。
- 不把复杂表格硬塞进一页，宁可拆页。
- 导出后逐张检查裁切，严禁标题/页码/二维码贴边。
